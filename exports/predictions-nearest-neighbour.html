Preprocessing

Target Variable

Machine learning algorithms work exclusively with numerical data and generate numerical outputs. Therefore, the first step is to ensure that our target variable is numeric. In our case, we need to categorize the temperature data to make it compatible with the Nearest Neighbors algorithm.

This categorization allows our model to use numerical labels instead of raw temperature values, making it suitable for the Nearest Neighbors approach while maintaining a structured representation of temperature ranges.

A new column named temp_category is created, which assigns a numerical category to each temperature value in the temperature_2m (C) column. The categorization follows these rules:

In[198]:
def categorize_temperature(temp):
    if temp < 5:
        return "Cold ( < 5C )"
    elif 5 <= temp < 15:
        return "Moderate (5 - 15C)"
    else:
        return "Warm ( >= 15C )"

data["temp_category"] = data["temperature_2m (C)"].apply(categorize_temperature)

Feature Selection

To optimize our model, we must select only those features that best explain the variance in the target variable. In our case, since we are categorizing temperatures, we need to identify which features have distinguishable patterns across different temperature categories.

For classification problems, a useful way to assess feature importance is by examining how well different features separate the temperature categories. A helpful visualization for this is a box plot, which can show the distribution of each feature across different temperature groups.

In[200]:
candidates = needed_columns[2:]
numeric_candidates = data[candidates].columns.tolist()

plot = data.boxplot(column=numeric_candidates, by="temp_category", figsize=(20,7), layout=(1,len(candidates)))

In this dataset, pressure (msl) (hPa) appears to be the best feature for predicting temperature, as its box plots show significant separation between different temperature categories. relative humidity (%) also demonstrates some separation and could contribute as a secondary feature. However, precipitation (mm) does not show strong separation and may be less useful for prediction.

Based on these findings, the selected features are stored in the variable X, and the target variable (temperature) is stored in y.

The scatter plot suggests that predicting temperature categories will be somewhat challenging. The Warm ( 15C) category is the most distinct, occupying a large portion of the lower pressure and lower humidity space, making it easier to classify. However, the Cold (< 5C) and Moderate (5 - 15C) categories have significant overlap, particularly in regions with higher humidity and higher pressure values. This indicates that the model may struggle to differentiate between these two groups due to their close proximity in feature space. While pressure and humidity are useful predictors, the moderate temperatures exhibit a more scattered distribution, which may introduce some classification difficulties in distinguishing borderline cases.
